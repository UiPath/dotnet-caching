# Caching Benchmarks With Aspire

Run benchmark smoke tests through the Aspire wrapper from the `Caching/`
folder. The wrapper starts `Sample.AspNetCore.AppHost` in the background, waits
for the Aspire Redis container to be reachable from the host, runs
`Caching.BenchmarkTests` on `net10.0`, then stops the AppHost process. The
Redis container uses a named volume so state survives across runs; clear it
explicitly (e.g. `docker volume rm`) for a fully cold benchmark.

Prerequisites:

- .NET SDK from `global.json`
- Docker Desktop or another Docker engine reachable by Aspire

## Default Benchmark Smoke

```powershell
.\eng\run-benchmarks-with-aspire.ps1
```

The default arguments run a short BenchmarkDotNet dry job:

```text
--filter *ReadKnownKeys* --job Dry --warmupCount 1 --iterationCount 1
```

Use `-NoBuild` after the solution has already been built:

```powershell
.\eng\run-benchmarks-with-aspire.ps1 -NoBuild
```

## Passing BenchmarkDotNet Arguments

Pass BenchmarkDotNet arguments after `--`:

```powershell
.\eng\run-benchmarks-with-aspire.ps1 -- --filter '*CacheBenchmark*'
```

Example full run for `ReadKnownKeys`:

```powershell
.\eng\run-benchmarks-with-aspire.ps1 -- --filter '*ReadKnownKeys*'
```

## Aspire Flags

The wrapper exposes the same AppHost modes as script parameters:

| Parameter | Behavior |
| --- | --- |
| `-UseShardedRedis` | Starts the Aspire sharded Redis topology and configures benchmark Redis endpoints from it. |
| `-UseSingleMachine` | Starts only `Machine1` in the AppHost. |
| `-NoOpenTelemetry` | Runs the sample without OpenTelemetry. |
| `-NoRedisInsight` | Skips Redis Insight resources. |
| `-StartupTimeoutSeconds <seconds>` | Changes how long the wrapper waits for Redis readiness. |

Shard-mode benchmark smoke:

```powershell
.\eng\run-benchmarks-with-aspire.ps1 -UseShardedRedis
```

Fast rerun after building:

```powershell
.\eng\run-benchmarks-with-aspire.ps1 -NoBuild -- --filter '*ReadKnownKeys*' --job Dry --warmupCount 1 --iterationCount 1
```

## Direct Benchmark Run

Prefer the Aspire wrapper for local benchmark runs because it supplies the Redis
connection string generated by Aspire. Direct `dotnet run` only works when Redis
configuration is already available through `Sample.AspNetCore/appsettings.json`
or environment variables:

```powershell
dotnet run --project Caching.BenchmarkTests/Caching.BenchmarkTests.csproj --framework net10.0 -c Release -- --filter '*ReadKnownKeys*'
```
